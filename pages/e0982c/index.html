# 一、FC组件传值未写入state（2021年01月12日）

问题描述：使用mock数据常量时正常，换为redux值或者state值后，传入有值但是未写入FC组件的state

失败解决方案：在useState后加上Object.assign触发更新

const [data, setData] = useState(value);
Object.assign(data, value) // 解决setState失效问题
1
2

但是这样会导致重复调用该组件时，传入的值只做后续assign，不是完全替换。

成功解决方案：使用useEffect.

useEffect(() => {
	// 解决动态初始值为空导致后续赋值失败的问题
	setData(value);
}, [value]);

# 二、keepalive功能添加（2021年03月03日）

antd pro项目框架，使用 react-activation 结合umi控制到页面级别的缓存。

####一、安装依赖 umi-plugin-keep-alive

通过依赖的安装，解决babel转义的问题

# 二、寻找适合放置 <AliveScope>

<AliveScope> 作为缓存域，需要放置在不会被卸载的位置

推荐放置在应用入口处，不过 umi 没有开放应用入口编辑权限，所以可以考虑放置在 layout 中

// 示例：/src/layouts/index.js
import { AliveScope } from 'react-activation';
import styles from './index.css';

function BasicLayout(props) {
  return (
    <AliveScope>
      <div className={styles.normal}>
        <h1 className={styles.title}>Yay! Welcome to umi!</h1>
        {props.children}
      </div>
    </AliveScope>
  );
}

export default BasicLayout;

# 三、详情页面时间选择反显（2021年07月24日）

时间选择器（年、日）在详情页面使用时，涉及到初始化反显问题。

<FormItem label="截止日期">
  {getFieldDecorator('date', {
    {/* 这里设置初始化赋值，会出现报错 */}
		initialValue: formValues.date,
		rules: [{ required: true, message: '请选择日期' }],
	})(<DatePicker placeholder="请选择日期" />)}
</FormItem>
1

从服务端取值设置时，会出现页面报错：

TypeError: value.locale is not a function
1

原因是未按照格式给DatePicker组件赋值。

改用下面的写法：


<FormItem label="截止日期">
  {getFieldDecorator('date', {
    {/* 格式化赋值 */}
		initialValue: formValues.date ? moment(formValues.date) : null,
		rules: [{ required: true, message: '请选择日期' }],
	})(<DatePicker placeholder="请选择日期" />)}
</FormItem>

const mergeCells = (text: string, array: any[], columns: string) => {
  let i = 0;
  if (text !== temp[columns]) {
    temp[columns] = text;
    array.forEach((item: any) => {

        i += 1;

# 十一、表格固定列时出现空白列问题

column使用fixed属性，table使用scroll的横向滚动属性后，如果column中每一列都使用width属性，则在屏幕足够宽时出现空白列。

解决方案：column中不全使用width属性，留下个别列（可加大scroll.x值，同时留一个较宽列做自适应，这样相当于有最小宽度兜底），自适应宽度即可。

PS：建议指定 scroll.x 为大于表格宽度的固定值或百分比。注意，且非固定列宽度之和不要超过 scroll.x。

# 十二、下拉框等交互赋值但UI未触发

2022年02月21日17:40:09

datepicker点击时未弹出选择框。下拉框选择option后输入框内容无变化，此时数据赋值等操作已成功，只是UI展示上异常。

原因：页面使用了KeepAlive。

范围：只对render中直接使用KeepAlive功能的页面有影响，如果是外层套了一层则无影响。
