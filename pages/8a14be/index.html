大屏适配问题总结

# 大屏适配

# 分辨率及适配

# 像素单位

# 设备像素Dp（dip）: Density-independent Pixels

以160dpi的屏幕为基准定义的，在160dpi的屏幕的屏幕上1dp=1px, 换算公式： 1dp =（屏幕ppi/ 160）px （这里px指物理像素）

# PPI：像素每英寸，表示每英寸所拥有的像素(pixel)数目。
# DPI：点每英寸，表示每英寸所拥有的点数。

在电子屏幕显示中，可认为 ppi 和 dpi相同

# 设备像素比（dpr： device pixel ratio ）：

公式：dpr = 同一方向上的 物理像素 / 逻辑像素 (缩放比是1的情况) DPR = 设备像素/CSS像素(设备独立像素 ) ~= DPI/160 window.devicePixelRatio // 获取dpr window.screen.width // 获取屏幕宽度

# 电视屏幕的页面尺寸到底是多少

# 大屏技术

# SVG

可缩放矢量图形（Scalable Vector Graphics），是基于可扩展标记语言（标准通用标记语言的子集）的用于描述二维矢量图形的一种图形格式。

# Canvas 2D

通过 JavaScript 来绘制 2D 图形，通过逐像素来进行渲染。

# Canvas 3D WebGL

WebGL（Web Graphic Library）是一个 JavaScript API，用于在任何兼容的 Web 浏览器中渲染 3D 图形。WebGL 程序由用 JavaScript 编写的控制代码和用 OpenGL 着色语言（GLSL）编写的着色器代码构成，这种语言类似于 C 或 C++，可在 GPU 上执行。

# 经验总结

# 内存泄露

  • 及时清除定时器（setTimeout、setInterval、requsetAnimationFrame）
  • 事件移除（resize）
  • 线上环境控制台console数据清空
  • 及时清除无用变量

# 数据处理

  • 注意防御式编程，对服务端接口的稳定性与可靠性持怀疑态度，定义默认值
  • 数据变动，先进行判断是否数据改变，之后再决定是否图表进行重绘，减少重绘次数
  • 特殊值测试
  • Echar中Dataset数据集的使用

# 布局

  • 可视化大屏基本是整屏展示，不需要滚动条，注意展示地点。
  • 动画部分最好使用css3动画，尽量少用canvas。 可以使用CSS的动画来启用硬件加速。CSS的  animations,  transforms 以及 transitions 不会自动开启GPU加速，而是由浏览器的缓慢的软件渲染引擎来执行。 能触发GPU加速的css属性： transform：translate(0)、transform:translate3d(0,0,0)、opacity、filter等等 另外，3D硬件加速提升动画性能时，最好给元素增加一个z-index属性，人为干扰复合层的排序，可以有效减少chrome创建不必要的复合层，提升渲染性能 PS. @keyframe必须写完整，否则移动端不起作用
  • 频繁更新的部分要使用绝对定位布局，脱离文档流，减少重排重绘。
