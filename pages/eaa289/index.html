前端打包及nginx配置

# hbuilder打包vue项目

# 需求

路由带前缀，便于多个项目放在同一域名下。要保证打包后通过nginx可以正常访问。

# 问题

使用history路由打包后，在非跟路由刷新时nginx出现404页面

# 尝试解决

使用hash路由，但是base依然写上前缀，如下所示：

"router" : {
  "mode" : "hash",
  "base" : "/my-app/"
},

# 问题原因

问题是由nginx配置问题导致的，因为location加了前缀xxx，所以 try_files 不能再使用/index.html而应该使用 xxx/index.html。

server {
  listen 1000;
  location /xxx {
    alias /build/address/;
    index index.html;
    try_files $uri $uri/ xxx/index.html; # 防止重刷新返回404
	}
}

# nginx配置接口转发

server {
  listen 1000;
  location /xxx {
    alias /build/address/;
    index index.html;
    try_files $uri $uri/ /xxx/index.html; # 防止重刷新返回404
	}
	# 注意^~的匹配模式，因此要将/xxx/也加入location保持一致。
	location ^~ /xxx/api/{
    proxy_buffering off;
    proxy_pass remote_api_url;
	}
}

其中proxy_pass后的url结尾是否带/会影响转发后的地址。

加/表示绝对根路径，不加表示相对路径，会将匹配的路径也带过去。

比如请求是http:127.0.0.1/api/interface

location /api/{
  proxy_pass http:127.0.0.1;
}
# 不加/, 转发后的地址为 http:127.0.0.1/api/interface
