H5截屏及屏幕录制解决方案

# H5截屏及屏幕录制解决方案

# H5截屏

# 开源包：html-to-image (opens new window)
# 原理：

将dom节点转换成canvans或svg，再生成图片文件。生成的文件包含base64-encoded的url，blob格式的文件流等格式。

# 交互方案支持：
  1. 前端对当前页面直接生成常用格式图片（svg、png、jpeg等）
  2. 前端对当前页面生成的文件流上传到服务端存储
  3. 其他需要进行二次加工的交互：如选择页面部分dom进行截屏

# H5屏幕录制

# 开源包：rrweb (opens new window)
# 原理：
  1. 录制：通过对事件的记录和页面快照的保存，将页面交互存储到eventlog中。该eventlog为数组格式的记录，一个动作可能包含数十条记录。
  2. 回放：通过 iframe 沙盒功能重建dom树，结合计时器将eventlog顺序执行。展现出类似于视频的效果。
# 交互方案支持：
  1. 控制开始录制和停止录制，将生成的eventlog存储。预览页面调用eventlog做回放。 订单贷开发服务器：白名单企业查询页面「开始录制」「结束录制」「重放操作记录」 地址：http://10.145.5.25:30020/ 账号：13261880001 密码：asd123123
  2. 使用开源包timecut (opens new window)（通过puppeteer将eventlog重放，再使用FFmpeg对回放进行录制）将eventlog生成mp4格式视频。演示视频：http://lab.myriptide.com/syncit-demo/record-2020-08-17_23.00.55.mp4
# PS：

由于使用 MutationObserver API，rrweb 不支持 IE11 以下的浏览器。谷歌、火狐、app内嵌webview基本都是支持的。
