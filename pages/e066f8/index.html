运管项目添加keep-alive

###运管项目添加keep-alive

####一、在layout组件中添加keep-alive

<transition name="fade-transform" mode="out-in">
  <!-- 方案一、使用inclue，完全匹配或者正则（正则匹配组件名包含KeepAlive） -->
  <!-- <keep-alive :include="/^[\w\W]*_KeepAlive$/"> -->
  <keep-alive include="EnterpriseUser_KeepAlive">
    <router-view/>
  </keep-alive>
  <!-- 方案二、需要在router的meta信息中添加keepAlive属性 -->
  <div>
    <keep-alive>
      <router-view v-if="$route.meta.keepAlive"></router-view>
    </keep-alive>
    <router-view v-if="!$route.meta.keepAlive"></router-view>
  </div>
</transition>

####二、keep-alive的include未生效

原因：项目中三处使用了<router-view/>，步骤一中只在layout组件中添加了keep-alive，另外两处分别是EmptyTemplate.vue和app.vue中。app.vue在最外层，不影响keep-alive的inclue，只需要在EmptyTemplate.vue中添加即可。

EmptyTemplate.vue是为了保证面包屑完整对路由做了深层嵌套。否则三级路由（详情页）由于只能和上级平级，面包屑也平级。如下所示：

####三、部分模块的组件在相互切换时keep-alive失效

问题描述：组件A和B均已keep-alive，但是从B回到A时，A的缓存失效。A到B时，B的缓存不失效。

原因：B组件未使用EmptyTemplate.vue组件包裹。路由层级不同且二级（B组件）大于三级（A组件），所以A的缓存失效。

解决：目前使用EmptyTemplate.vue包裹住组件B，保持层级一致。

####四、问题三的解决方案

======> 解决方案：按照正常router父子层级嵌套，即都在layout下，无更深层级。

具体实现：

  1. 去除EmptyTemplate.vue组件包裹，直接使列表页和详情页面同级，

  2. 面包屑问题通过修改渲染方式及title定义来实现。

// 列表及详情组件路由写法

  meta: {
    title: ['融资企业注册用户查询', '用户详情'] // 深层级的面包屑用title
